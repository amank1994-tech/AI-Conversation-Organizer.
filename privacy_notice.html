Privacy Notice · Version 2.1

What this extension does
with your data

AI Conversation Organizer · Last updated March 2026
This extension does not violate your privacy — here is why
  • Zero data ever leaves your device. The extension makes no network requests of any kind. Nothing is sent to any server, ever. You can verify this yourself in Chrome DevTools → Network tab.
  • Your message text is never stored. The extension reads your messages locally to detect topic categories (like "Health" or "Career"), then immediately discards the text. Only the category label — a single word — is saved.
  • AI responses are never read. The extension only looks at messages you wrote (marked by ChatGPT as role="user"). It never touches any AI-generated text.
  • One permission. That's all. The extension only requests storage — to save your conversation list on your own device. No tabs, no history, no clipboard, no identity.
  • You can delete everything anytime. Individual conversations or all data can be wiped from inside the extension. Uninstalling removes everything permanently.
Important — what changed in version 2.1
Earlier versions only read the conversation title (from the browser tab). Version 2.1 added multi-topic detection, which means the extension now also reads the text of messages you write inside a conversation. It does this entirely on your device, instantly discards the text, and stores only a category label like "Health" or "Career". No text you have ever typed is retained or transmitted. This notice explains exactly how that works.

How topic detection works — step by step

When you are in a ChatGPT conversation, here is the exact sequence of what happens inside the extension:

Data flow — local processing only
1
You send a message to ChatGPT
ChatGPT renders your message in the page DOM with the attribute data-message-author-role="user".
2
The extension detects a new message was added to the page
A MutationObserver notices that a new element appeared in the conversation. It waits 1.2 seconds (debounce) before doing anything, so it does not fire during streaming.
3
Your message text is read from the DOM
The text of every user message in the conversation is read into a local JavaScript variable. This happens entirely inside your browser — nothing is sent anywhere.
⚠ Text exists in memory at this point
4
Keyword matching runs against a fixed list
The text is lowercased and checked against a hardcoded list of category keywords — e.g. if the text contains "skincare" or "acne", the label "Health" is returned. If it contains "resume" or "interview", the label "Career" is returned. The full list is visible in the source code.
5
The text variable is discarded
The function returns only the matched label names (e.g. ["Health", "Career"]). The text string goes out of scope and is garbage-collected. It is never written to storage, never passed to another function, and never transmitted.
Text discarded immediately
6
Only the category labels are saved
The label strings (e.g. "Health", "Career") are merged into the conversation record in chrome.storage.local on your device. This is the only thing written to storage from this entire process.
✓ Only label names stored
You can verify this yourself
Open Chrome DevTools (F12) while the extension is active. Go to the Network tab — you will see zero requests made by the extension. Go to Application → Storage → Local Storage — you will see only the category label strings, never any message text. The full source code is available for inspection in the extension folder.

Your rights and controls

Delete individual conversations: Click the × button on any conversation card. It is removed from local storage immediately.

Delete everything: Click the trash icon in the top-right of the extension and confirm. All data is permanently erased from your device.

Uninstall the extension: Go to chrome://extensions, find AI Conversation Organizer, and click Remove. Chrome deletes all associated local storage when an extension is uninstalled.

See exactly what is stored: Open DevTools → Application → Local Storage → chrome-extension://[id] → key aco_conversations. You will see the raw JSON with every field — confirming that no message text is present.

Third-party sharing

The Share feature — when you click the share icon on a conversation card — is entirely optional and user-initiated. If you choose to share, the conversation URL is passed to the standard share page of whichever platform you select (Twitter/X, LinkedIn, or WhatsApp). This only happens when you explicitly click a share button. Nothing is shared automatically.

No data is shared with the developer of this extension, with Anthropic, with OpenAI, or with any other third party under any circumstances.

Compliance with privacy laws

GDPR (EU/UK): Under GDPR, "processing" personal data requires a legal basis. The processing here — local keyword detection of user-authored messages — is performed at the user's explicit request (by installing and using the extension), produces no identifiable output, and involves no data transfer outside the user's own device. This falls under the "legitimate interests" and "performance of a contract" bases. No data leaves the EU or the user's device.

CCPA (California): The extension does not "sell" or "share" personal information. It does not collect data for advertising. The user's information is used solely to provide the feature they requested.

PIPEDA (Canada) / PDPA (various): Similar principles apply. All processing is local, user-initiated, and produces no persistent personal data beyond what the user can see and delete at any time.

Note: Because all data is stored locally on the user's device and no data is transmitted, the developer of this extension is not a "data controller" or "data processor" in the traditional sense under most privacy frameworks. The user is effectively processing their own data on their own device.

Children's privacy

This extension is not directed at children under 13. It does not knowingly process data from children. If you are under 13, please do not install this extension.
